CaseProof Analyst

A terminal-first DFIR agent for FIND EVIL: it opens real disk evidence read-only, runs typed SIFT-compatible tools, verifies claims, drops unsupported conclusions, and writes a traceable evidence package.

Input: real E01 disk image Runtime: WSL SIFT-compatible tools Output: reports + evidence book + logs UI: terminal and generated files

What this project is

This is not a normal website. It is a forensic triage tool. The user gives it a disk image, and the project creates a defensible first-pass case package.

Plain version: it checks a suspicious Windows disk image and refuses to say "compromised" unless the evidence actually supports that claim.

What it does in one run

1

Open evidence

Registers base-rd-01-cdrive.E01 as read-only input and verifies SHA256.

2

Find volume

Uses SIFT-compatible tools to identify an analyzable NTFS volume boundary.

3

Inventory artifacts

Finds registry hives, event logs, user hives, and McAfee event paths.

4

Verify claims

Confirmed findings need evidence references. Unsupported claims are not promoted.

5

Write package

Creates analyst report, evidence book, correction ledger, accuracy report, and execution log.

Self-correction moment

The tool deliberately rejects this tempting claim:

Original candidate: Confirmed compromise or persistence on RD01.
Reason challenged: unsupported_claim
Final status: dropped
This is the point of the project: it finds useful artifacts quickly, but it does not upgrade "artifact exists" into "machine is compromised" without content-level proof.

Command that creates the result

py scripts\run_real_case.py --case-id CASE-RD01-DEMO --evidence-path evidence\base-rd-01-cdrive.E01 --case-workspace cases\CASE-RD01-DEMO --json

This command is what you should record in the demo video. It creates a fresh case package from the real evidence.

What it does not do yet

  • It does not claim full incident reconstruction.
  • It does not claim a confirmed malicious finding.
  • Registry, event, and timeline content-level parsing remain future work.
  • The project is terminal-first; this HTML page is only a visual explainer.
